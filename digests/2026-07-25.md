## 1. 「Beyond the Leaderboard」— LLMエージェントの失敗を6クラスターに体系化した大規模サーベイ論文

**情報源:** [arXiv:2607.05775](https://arxiv.org/abs/2607.05775)（Wael Albayaydh, Rui Zhao, Ivan Flechais、2026年7月7日投稿、cs.AI）

**何が新しいか:**
2023〜2026年に発表された27本のベンチマーク・分類・監査論文（19種類のベンチマークを横断）を統合し、LLMエージェントの限界を単一の分類体系にまとめたサーベイ。個別ベンチマークのリーダーボード数値だけでは見えない「なぜ失敗するか」という共通パターンを横断的に整理した点が新しい。

**技術的なポイント:**
- 失敗を6つのクラスターに分類：(1) ツール呼び出し・パラメータレベルのエラー、(2) 計画立案・制約充足の失敗、(3) コンテキスト蓄積による長期タスクでの性能劣化、(4) マルチエージェント間の協調失敗、(5) 敵対的・未規定条件下での安全性・セキュリティ上の失敗、(6) 測定妥当性（ベンチマーク自体の信頼性)の問題。
- 単一のスコアでは捉えられない「タスクは成功したが途中のツール呼び出しは誤っていた」のような部分的失敗パターンにも言及。
- 16ページ、3表・1図の構成で、既存19ベンチマークの弱点も横断比較。

**なぜ重要か:**
エージェントを設計・評価する際、単体ベンチマークのスコアだけを見ると見落とすエラーの型が具体的に列挙されている。FDEとして顧客環境にエージェントを導入する際、「どこで壊れやすいか」を先回りして説明・対策できるようになる実務的価値が高い。

**リンク:** https://arxiv.org/abs/2607.05775

## 2. DeepSeek V4がプレビューからGA(一般提供)へ — 1Mコンテキスト・新価格体系、旧APIは7/24で廃止

**情報源:** [DeepSeek API Docs](https://api-docs.deepseek.com/news/news260424/)（V4プレビュー発表の公式ページ）、[kie.ai](https://kie.ai/blog/deepseek-v4-release-what-we-know)、[tech-insider.org](https://tech-insider.org/au/deepseek-v4-general-availability-2026/)（GA移行の報道、2026年7月20日前後）※WebFetchが遮断されたため複数の独立した二次情報源で内容が一致することを確認

**何が新しいか:**
2026年4月24日にプレビュー公開されたDeepSeek V4が、7月20日付けでGA（一般提供）に移行。旧モデル名`deepseek-chat`/`deepseek-reasoner`は7月24日15:59 UTCで完全に廃止され、新エイリアスへの移行が必須になった。

**技術的なポイント:**
- ラインアップは2種：`DeepSeek-V4-Pro`(総パラメータ1.6T、活性化49B/token のMoE)と`DeepSeek-V4-Flash`(総パラメータ284B、活性化13B/token)。いずれもオープンウェイトでHugging Face公開。
- 両モデルとも1Mトークンのコンテキスト長に対応。GA版ではエージェント向け機能・数学推論・コード生成が強化されたとアナウンスされている。
- 新たに「ピーク/オフピーク」の時間帯別価格体系を導入。入力トークン最安値は100万トークンあたり$0.14程度と報じられている（価格・ベンチマーク数値は二次情報源の報道であり、DeepSeek公式のベンチマークカードで直接確認はできていない点に注意）。

**なぜ重要か:**
オープンウェイト・低価格・1Mコンテキストという組み合わせは、自前でエージェント基盤を構築する際のコスト試算に直結する。既存コードで`deepseek-chat`等のモデル名をハードコードしている場合、廃止期限を過ぎるとAPI呼び出しが失敗するため、移行要否を早めに確認する価値がある。

**リンク:** https://api-docs.deepseek.com/news/news260424/

## 3. GitHub Code Quality、プレビューから正式な有償製品へ — $10/コミッター/月、AI機能は従量課金

**情報源:** [GitHub Changelog](https://github.blog/changelog/2026-06-16-github-code-quality-generally-available-july-20-2026/)（2026年7月20日付でGA、告知は6月16日）※WebFetchが403で遮断されたため、検索結果に表示された同URL・同ドメインの記載内容で実在と概要を確認

**何が新しいか:**
GitHub Code Qualityが2026年7月20日、パブリックプレビューから購入可能な正式製品としてGA（一般提供）に移行した。

**技術的なポイント:**
- 有効化したリポジトリに対し、アクティブなコミッターあたり月額$10の課金体系。
- Copilotによるコードレビュー、AIによる問題検知、Copilot Autofixなど、AI機能部分は従量課金（usage-based）で別途加算される仕組み。
- 同時期にGitHubはPull Requestsダッシュボードの刷新やCopilotメトリクス（採用状況・利用トレンドの可視化）ダッシュボードなど、Copilot活用度を計測・可視化する機能群を強化している。

**なぜ重要か:**
「AIコードレビュー/Autofixの効果を定量化して経営層に説明する」というのは、AIエージェント導入を推進する立場（FDE的な役割）で頻繁に求められる仕事そのもの。価格モデルや計測指標の設計を知っておくと、顧客への提案時に説得力のある材料になる。

**リンク:** https://github.blog/changelog/2026-06-16-github-code-quality-generally-available-july-20-2026/
